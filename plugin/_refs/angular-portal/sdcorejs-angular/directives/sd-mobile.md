# `*sdMobile` Directive

**Type**: Structural Directive
**Selector**: `[sdMobile]`
**Class**: `SdMobileDirective`
**Standalone**: yes (Angular 19 default — no `standalone: true` flag needed)
**Library version**: `@sd-angular/core@19.0.0-beta.86`

**Import path**: `@sd-angular/core/directives` (or direct: `@sd-angular/core/directives/sd-mobile`)

## One-line purpose
Structural directive that renders its host template ONLY on mobile viewports, evaluated once at construction time.

## When to use
- Show a mobile-only menu trigger / hamburger / bottom-nav button that does not exist on desktop
- Render a compact mobile variant of a content block (typically paired with `*sdDesktop`)
- Hide controls that desktop already renders inline (e.g. inline action row → floating action button on mobile)
- Conditionally include a mobile-specific instruction / empty-state message

## When NOT to use
- For viewport-reactive UI — evaluated ONCE at construction; does not respond to orientation change or window resize. Use CDK `BreakpointObserver` or CSS media queries for reactive behavior.
- For minor responsive tweaks (spacing, font size) — prefer CSS media queries; no need to fork the DOM.
- Inside a component that is mounted only once per session but whose viewport context changes (e.g., a panel that opens in a responsive iframe) — evaluate `isMobile()` dynamically with `*ngIf="isMobile()"` instead.
- As a replacement for accessibility-safe visibility — a hidden template is never instantiated, so screen readers cannot reach its content at all. Consider whether the content should be hidden or simply styled differently.

## Inputs
None — visibility is decided entirely by `BrowserUtilities.isMobile()` at construction time.

## Outputs
None.

## Content projection
N/A — structural directive; operates on `ng-template` generated by Angular's `*` desugaring. Does not project content into a slot.

## Visual cues
N/A — directive does not render any host element or apply host classes; it either creates or skips the embedded view entirely.

## Behavior / Lifecycle

### Host bindings
None — the directive adds no host element, no CSS class, and no attribute.

### Lifecycle
- **constructor-only**: calls `BrowserUtilities.isMobile()` exactly once.
  - Returns `true` → calls `viewContainerRef.createEmbeddedView(templateRef)` to instantiate the template.
  - Returns `false` → template is never instantiated; no DOM node is created.
- No `ngOnInit`, `ngOnChanges`, or `ngOnDestroy` hooks. Behavior is fixed after construction.

### Side-effects
- **Render**: if mobile, the full template subtree is created and attached to the view container.
- **Skip**: if desktop, zero DOM nodes are created; Angular lifecycle hooks inside the template are never called.
- **NOT reactive**: no subscription, no event listener on `window.resize` or `window.orientationchange`. If the user rotates the device after the component has mounted, the directive does NOT re-evaluate.

## Examples

### 1. Mobile-only hamburger trigger
```html
<sd-button
  *sdMobile
  type="link"
  prefixIcon="menu"
  (click)="onOpenDrawer()">
</sd-button>
```

### 2. Mobile-specific empty-state copy
```html
<div class="empty" *sdMobile>
  <p>Vuốt sang phải để xem thêm tùy chọn.</p>
</div>
```

### 3. Pairing with `*sdDesktop` for layout variants
```html
<!-- Mobile: stacked card list -->
<ng-container *sdMobile>
  <app-mobile-card-list [items]="items" />
</ng-container>

<!-- Desktop: data table -->
<ng-container *sdDesktop>
  <app-data-table [items]="items" />
</ng-container>
```

### 4. Wrapping a component tree that should only exist on mobile
```html
<ng-container *sdMobile>
  <div class="bottom-nav">
    <a routerLink="/home">Home</a>
    <a routerLink="/search">Search</a>
    <a routerLink="/profile">Profile</a>
  </div>
</ng-container>
```

## Anti-patterns
- **Stacking `*sdMobile` with `*ngIf` on the same element** — Angular does not support two structural directives on the same host. Wrap in `<ng-container>`:
  ```html
  <!-- WRONG -->
  <div *sdMobile *ngIf="show">...</div>

  <!-- CORRECT -->
  <ng-container *sdMobile>
    <div *ngIf="show">...</div>
  </ng-container>
  ```
- **Using for typography / spacing differences** — prefer CSS media queries; avoid forking the DOM for style-only changes.
- **Relying on `*sdMobile` for content that must update when orientation changes** — the directive evaluates `isMobile()` once at init, not reactively. When `isMobile()` would return a different value after the component is mounted (e.g., user rotates device), the template is NOT re-rendered. If dynamic behavior is required, use `*ngIf="isMobile()"` combined with a `window.resize` listener, or `BreakpointObserver` from `@angular/cdk/layout`.

## Related
- `*sdDesktop` — the inverse: renders only when `isMobile()` returns `false`.
- `BrowserUtilities.isMobile()` — the underlying UA-string detection helper (`/Mobi|Android/i.test(navigator.userAgent)`).
- `BreakpointObserver` (`@angular/cdk/layout`) — reactive alternative when the directive's one-shot evaluation is insufficient.
