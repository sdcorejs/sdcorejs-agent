# SdLicenseService

**Type**: Service (Angular `@Injectable`)
**Class**: `SdLicenseService`
**Provided in**: `'root'`
**Import path**: `@sdcorejs/angular/services/license`
**Library version**: `@sdcorejs/angular@20.0.1`

## One-line purpose
Domain-binding license check that hashes `window.location.hostname` (with a built-in salt) and compares against `licenseKey` values from `SD_CORE_CONFIGURATION`; throws on unauthorized domains and bypasses verification on localhost.

## When to use
- Licensed/private Angular libraries built on `@sdcorejs/angular` that should only run on contracted customer domains.
- Apps that already provide `ISdCoreConfiguration` and want a hard runtime guard before exposing premium features.
- Boot-time enforcement: call `enforceLicense()` from `APP_INITIALIZER` to block bootstrap on unauthorized hosts.

## When NOT to use
- For per-feature flags / role-based authorization — use a real authZ service. This is a domain-level kill switch, not a permission system.
- For production secrets — the salt is in client code; this guards against casual misuse, not a determined attacker.
- In tests — mock the service; the constructor calls `#verify()` immediately and may throw.

## Public API

### `enforceLicense(): void`
Throws if the current hostname did not pass `#verify()` during construction. Safe to call repeatedly.

```typescript
enforceLicense = (): void;
```

**Throws**: `Error('[Security] Unauthorized usage of UI Lib on domain: <hostname>')` when the hostname is not localhost and does not match any configured exact or wildcard key.

## Configuration / DI tokens

### `SD_CORE_CONFIGURATION` — provides `licenseKey`
The service reads `coreConfiguration.licenseKey` (a single hash string or an array of hash strings) from `@sdcorejs/angular/configurations`.

```typescript
// app.config.ts
providers: [
  { provide: SD_CORE_CONFIGURATION, useValue: <ISdCoreConfiguration>{
    licenseKey: ['NTA0NDM2NjE3OTk0c2lnbmVk', 'LTYzMjMyOTkyMDdzaWduZWQ='],
    // ... other core config
  }},
]
```

Each key is generated by hashing `<domain> + SALT` with the service's internal algorithm (a 32-bit string hash, then `btoa(hash + 'signed')`). Keys are normally produced by a build-time tool the lib vendor provides — do not try to compute them at runtime.

## Behavior notes
- **Localhost bypass**: `localhost`, `127.0.0.1`, or any hostname containing `'localhost'` is auto-approved (logs a dev-mode banner to console).
- **Verification flow**:
  1. If no `licenseKey` is configured → throw immediately.
  2. Try exact match: `hash(hostname + SALT)` against the key list.
  3. Wildcard fallback: progressively shift subdomains and try `hash('*.' + suffix + SALT)`. Stops at TLD+1 (won't match `*.com`).
- **Salt** is a hard-coded constant in source: `'angular-core-976e2fa6f8b44dadbc63f87b057a331f'`.
- **Side effect at construction**: the constructor calls `#verify()`. If verification fails it throws — i.e. injecting the service alone can throw. `enforceLicense()` rethrows the same condition.
- **Security caveat**: the hash is a non-cryptographic 32-bit string hash (`(hash << 5) - hash + char`). Do not treat keys as secrets — they only block trivial domain swaps.

## Examples

### 1. Boot-time guard with `APP_INITIALIZER`
```typescript
import { APP_INITIALIZER } from '@angular/core';
import { SdLicenseService } from '@sdcorejs/angular/services/license';

providers: [
  {
    provide: APP_INITIALIZER,
    useFactory: (lic: SdLicenseService) => () => lic.enforceLicense(),
    deps: [SdLicenseService],
    multi: true,
  },
],
```

### 2. Configure single-domain license
```typescript
{ provide: SD_CORE_CONFIGURATION, useValue: {
  licenseKey: 'NTA0NDM2NjE3OTk0c2lnbmVk',     // hash for 'app.customer.com'
}}
```

### 3. Configure wildcard subdomain
```typescript
// Generated from '*.customer.com'
{ provide: SD_CORE_CONFIGURATION, useValue: {
  licenseKey: ['LTYzMjMyOTkyMDdzaWduZWQ='],
}}
// Now app1.customer.com, app2.customer.com all pass via wildcard fallback.
```

### 4. Guard a feature lazily
```typescript
class PremiumFeature {
  constructor(private lic: SdLicenseService) {}
  open() {
    this.lic.enforceLicense();   // re-check before showing
    // ... open premium UI
  }
}
```

## Testing

### In unit / integration tests
The constructor runs `#verify()` immediately. On `localhost` (Karma / Jest default) the bypass fires and the service is safe to inject. No mock needed for localhost-only tests.

```typescript
// Simplest — just inject: localhost bypass always passes in Karma
TestBed.configureTestingModule({});
const service = TestBed.inject(SdLicenseService);
expect(() => service.enforceLicense()).not.toThrow();
```

For tests that must exercise non-localhost paths, provide a pre-computed hash via `SD_CORE_CONFIGURATION` **and** spy on `window.location.hostname`. Because `window.location` is read-only in many browser environments, use:

```typescript
spyOnProperty(window, 'location').and.returnValue({
  ...window.location,
  hostname: 'app.example.com',
} as Location);
```

If the browser sandbox disallows the spy, mark those tests as `pending()`.

To completely skip license enforcement in tests, provide a stub:

```typescript
{ provide: SdLicenseService, useValue: { enforceLicense: () => {} } }
```

## Anti-patterns
- Do NOT ship the SALT or generated keys as if they were security tokens — they're an obfuscation, not authentication.
- Do NOT call `enforceLicense()` inside hot loops — it just re-checks a boolean, but the throw is uncaught and will crash the zone.
- Do NOT add `localhost` to your `licenseKey` list — it already bypasses, and the dev banner won't show.
- Do NOT compute keys in app code with `btoa(...)` of arbitrary strings — the vendor's hash algorithm must match exactly.
- Do NOT forget that the `SD_CORE_CONFIGURATION` token is `@Optional()` — omitting it is not an error on localhost, but will throw on any real hostname.

## Related

- `APP_INITIALIZER` (Angular) — recommended way to fail-fast on bootstrap.
